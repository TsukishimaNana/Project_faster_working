# 个人全局配置

## 回答偏好
- 回复使用中文
- 代码修改前先简要说明修改思路，不要直接给出代码
- 遇到有多种实现方案时，列出选项让我选择，而不是直接选一种

## 通用约定
- 提交信息使用英文，格式：`type(scope): description`
- 新文件开头不加版权注释
- 优先使用原生 API，避免引入不必要的依赖
- 本文件只记录个人表达偏好，不触发重工程流程；仓库默认执行方式以根目录 `AGENTS.md` 的 Demo 优先模式为准。

## 文档语言策略
- 项目文档优先使用“中文主体 + 英文标识”的格式，避免全英文文档增加人工阅读成本
- 面向人阅读和验收的文档使用中文为主，例如 `README.md`、`Handoff.md`、`agents.md`、`docs/*.md`
- OpenSpec 文档允许中英混合：文件名、change id、Requirement 名、命令、路径和接口标识保留英文，需求说明、验收标准、风险和领域语义优先使用中文
- 代码相关标识保持英文，例如包名、函数名、CLI 命令、配置字段、输出文件后缀和测试名
- 高风险规则必须用中文写清楚，包括原始文件只读、中文路径/乱码处理、SVG 中文文字层、PDF 中文字体、source-of-truth 判断、SVG 不是最终目标、不能过度圆顺关键纸样特征

## 安全习惯
- 修改认证相关代码前主动提示我注意安全影响
- 不要在代码注释或日志中输出任何密钥或 token
