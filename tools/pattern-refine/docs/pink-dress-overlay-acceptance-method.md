# 粉色连衣裙 PDF/SVG 合格对照分析方法

本文是给后续 AI 助手和实现者使用的验收方法说明。它不是算法实现说明，而是先定义
`pink-dress-original-scan.pdf`、人工勾线 SVG、以及合格叠图之间应该满足的关系。

## Source of Truth

只读参考文件：

- 原始扫描 PDF：`knowledge_base/PDF-SVG/Original_PinkShirts/pink-dress-original-scan.pdf`
- 人工勾线 SVG：`knowledge_base/PDF-SVG/Original_PinkShirts/pink-dress-simple-reference.svg`
- 合格叠图 PDF：`knowledge_base/PDF-SVG/Original_PinkShirts/pink-dress-original-scan-SVG-VS-PDF.pdf`

`pink-dress-original-scan.pdf` 是原 PDF。`pink-dress-simple-reference.svg` 是人工勾线结果，不是
raw tracing。合格结果应接近 `pink-dress-original-scan-SVG-VS-PDF.pdf` 中红色矢量线与扫描底图的
叠合关系。

## 必须先处理页面方向

这是已经反复犯过的验收错误，必须作为第一检查项。

- 原始 PDF 的第一页 `page.rect` 是约 `1190.52 x 842.04 pt`，`rotation=270`。
- 合格叠图 PDF 是约 `842.04 x 1190.52 pt`，`rotation=0`。
- 两者物理尺寸都是 A3：`297.053 x 419.989 mm` 或旋转后的同一页面。
- 人工 SVG 的 `viewBox` 是 `0 0 842 1190.5`，与合格叠图 PDF 的竖版坐标一致。

因此任何比较前都要把 candidate、scan render、reference SVG 归一到同一页面方向：

1. 先检查 PDF `rotation`、`page.rect`、render 后图像方向和 SVG `viewBox`。
2. 如果整体 overlay 大面积错位，先尝试 0/90/180/270 度旋转和必要镜像。
3. 如果 90 度旋转能让大部分裁片位置和朝向接近，先修页面方向归一化，不要继续调 path matching、smoothing 或 feature rules。
4. 结构指标 `9 path / 10 line / 1 rect` 不能替代方向检查。

## 合格红线的含义

合格叠图中的红线是生产纸样几何，不是扫描墨迹外轮廓：

- 红线应落在黑色扫描线条的中心或设计意图位置。
- 红线是 single centerline-like stroke，不是黑色墨迹 blob 的外边界。
- 红线会跨过扫描里的轻微断裂，按纸样意图连接成连续边。
- 红线会忽略线宽、抖动、灰尘、手写、箭头、服装示意图、页面边框和无关噪声。
- 红线会保留剪口、尖点、短对位标、直角和必要折点。

如果输出出现双边线、边缘重线、沿墨迹两侧跑、局部断线，即使对象数量正确，也是不合格。

## 比例尺处理

比例尺是 1:1 输出的 source-of-truth 候选，不能用整条黑色延长线。

合格叠图中比例尺表达为两组独立 `line` 对象：

- 水平比例尺主线长度约 `85 pt = 29.99 mm`。
- 竖直比例尺主线长度约 `85 pt = 29.99 mm`。
- 每组有 4 个刻度位置，对应 `0mm / 10mm / 20mm / 30mm`。
- 相邻刻度间距约 `28.3 pt = 9.98 mm`。
- 刻度线本身约 `3.5-4.0 pt = 1.23-1.41 mm`。

正确检测模型：

1. 找到一条近水平或近垂直的主线。
2. 只截取有 4 个连续刻度的可读段。
3. 验证 3 个间隔近似等距，每格代表 `10mm`。
4. 该可读段总长定义为 `30mm`。
5. 主线后方没有刻度的黑色延长线不能计入 30mm。

错误检测模型：

- 只用 Hough 找最长直线。
- 把有刻度 30mm 段和后方黑色延长线合并。
- 选中 unrelated 长线作为比例尺。
- 发现 marker 与 page-box 有差异时直接静默改 scale。

scale report 必须说明：

- 检测到的 4 个刻度坐标。
- 三个 10mm 间隔的像素距离。
- 最终 30mm 可读段的起点、终点、方向。
- `measured_pixel_to_mm` 与 `pdf-page-box` 的差异。
- 当前导出实际采用哪个 scale source。

## 弧线处理

人工 SVG 的弧线不是逐像素拟合扫描黑线，而是按制版意图重绘：

- 长弧线应平滑、连续，用 Bezier 曲线表达。
- 扫描抖动、小波浪和线宽变化不应进入最终曲线。
- 曲线可以不完全压在黑色像素边缘上，但应落在纸样边的中心趋势线上。
- 弧线的端点必须与相邻直线、尖角或剪口连接，不应留下断缝。
- 弧线圆顺不能吞掉剪口、短对位标、尖角和直角。

分析弧线时不要问“是否贴合每个黑像素”，而要问：

- 红线是否表达同一条生产边。
- 是否连续。
- 是否通过黑色扫描线的中心趋势。
- 是否保留了边上的生产特征。

## 直线处理

人工 SVG 中直线用于表达明确的生产直边、矩形条、比例尺和部分 construction edge：

- 直线边应保持直，不应被曲线拟合成波浪。
- 扫描中的轻微弯曲或手抖不应让直边变成折线碎片。
- 长方形条应输出为 `rect` 或等价直线矩形语义，不应作为墨迹轮廓碎 path。
- 比例尺必须输出为独立 `line` 对象，不应混入裁片 path。
- 对位短线可独立保护；不要当成噪声删掉。

判断直线的依据不是局部像素边界，而是整体边的设计趋势和 reference 中的语义对象类型。

## 尖角、剪口和 notch

尖角和剪口是生产特征，优先级高于圆顺：

- 尖角必须保持尖，不允许 smoothing 把它圆掉。
- V 形或三角形 notch 要保持角状。
- 边上的短 protruding/recessed mark 要保留。
- 直角和 near-right-angle construction corner 要保留。
- 在曲线拟合前先识别并冻结这些保护点。

如果某处黑线因为扫描断裂导致 notch 边缘不完整，人工 SVG 仍会按纸样意图画出完整尖点。
因此自动算法应倾向于“重建生产特征”，而不是忠实保留断裂像素。

## 合格对照的检查流程

每次输出 SVG/PDF 后，按以下顺序检查：

1. **页面方向检查**
   - candidate 是否和 reference 同为 A3 竖版坐标。
   - 是否存在明显 90 度旋转或镜像。

2. **比例尺检查**
   - 是否检测到 4 个连续刻度。
   - 是否只取 30mm 可读段。
   - 每格是否是 10mm。
   - 是否没有把尾部延长线计入比例尺。

3. **结构检查**
   - 参考目标大致为 `9 path / 10 line / 1 rect`。
   - 结构通过只表示对象数量接近，不代表几何合格。

4. **整体叠图检查**
   - 红线是否在扫描纸样线中心趋势上。
   - 是否排除了手写、箭头、服装示意图和噪声。
   - 是否没有双边线、重线、断线。

5. **逐裁片检查**
   - 每个裁片是否方向一致。
   - 长弧线是否平滑连续。
   - 直线边是否直。
   - notch、尖角、直角和短对位标是否保留。

6. **导出 PDF 检查**
   - PDF 必须是矢量输出。
   - 不应嵌入 raster image 作为最终线稿。
   - 线宽、坐标单位和页面尺寸应保持 1:1。

## 禁止把这些当作通过

以下情况必须判为未通过：

- 只因 object count 接近 reference 就判定 MVP pass。
- 比例尺取了无刻度延长线。
- 输出是墨迹外轮廓，形成双边线。
- 裁片边缘有明显断线。
- 整体 overlay 显示 90 度旋转或镜像仍未处理。
- smoothing 后尖角、剪口、notch 被圆掉。
- 使用 `cleaned.svg` 的低质量轮廓作为唯一 source-of-truth，不回退到 candidate/raster 重新重建。

## 给 AI 助手的执行提示

后续 AI 助手继续 MVP 时，先读本文，再动代码。特别注意：

- 先分析合格叠图关系，再改算法。
- 先修全局方向和比例尺，再修局部轮廓。
- 不要再把结构 gate 当成生产验收。
- 当前主要技术缺口是从扫描墨迹重建中心线式生产几何，而不是继续微调外轮廓简化参数。
